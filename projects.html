Chord

Chord is a project that implements the P2P distributed system introduced by various researchers from MIT in 2001. Chord uses a distributed hashtable to assign keys to different nodes in a network and by doing so allows clients to share a vast amount of data, which is maintained by all the nodes in the system, while only holding a small amount of data themselves. All of this is done without the need of a single main server because each node acts as both client and server. To test this program, run the Chord executable and more instruction is provided. To see the full fuction of this program, I recommend running the program in multiple terminals to create a small Chord Ring.

RPC-Chat

This program provides a very limited interface and is mostly used to demonstrate how Remote Proceedure Calls work. In practice we would likely use an RPC package to do this work, but this code provides insight into what happend what an RPC call is made such as serializing data and sending it across a network to call a method on another machine. To run this program, run the executable along with some specified arguments. If starting up the first node in the system give a port number for the server to listen on. Otherwise, give the address of the server and then a username to login on.

Fractal

This Project contains a lot of code and functionality that I wont get into. I recommend simply testing it because the end product is pretty cool. Run the glut_main executable in the gui-src folder. The default display is a Mandelbrot fractal. By pressing 'n' you can switch it to a Julia fractal, 'b' will switch back. By clicking somewhere on the Mandelbrot fractal, the Julia Fractal will change (best results are from clicking on the edge of the mandelbrot).
